# Agent Instructions

## Required First Step

Перед началом любой задачи агент обязан:

1. Открыть и прочитать файл `RULES.MD`
2. Убедиться, что решения соответствуют правилам проекта
3. Учитывать архитектуру проекта и текущую структуру папок

---

## Priority

1. Системные ограничения среды
2. Явный запрос пользователя
3. RULES.MD
4. Локальные технические решения агента

---

## Project Context

Проект представляет собой веб-приложение для пауэрлифтинга:

* Выбор тренировочных программ
* Расчёт рабочих весов на основе 1ПМ
* Отображение тренировочного процесса
* Аналитика и прогресс пользователя

Технологии:

* Next.js (App Router)
* TypeScript
* Zustand (state management)
* CSS Modules / SCSS

---

## Mandatory Behavior

Агент обязан:

1. Не изменять архитектуру проекта без явного запроса пользователя
2. Не писать код, противоречащий RULES.MD
3. Перед генерацией кода проверять:

   * где должен лежать файл
   * к какой части проекта относится логика (ui / feature / store / lib)
4. Давать краткое объяснение перед сложными решениями
5. Писать чистый, читаемый и типизированный код
6. Избегать лишней абстракции и переусложнения

---

## Code Generation Rules

При генерации кода:

* Использовать TypeScript (никакого `any` без причины)
* Следовать существующей структуре проекта
* Не дублировать уже существующую логику
* Проверять, можно ли переиспользовать существующие компоненты или функции
* Не создавать новые глобальные зависимости без необходимости

---

## State Management (Zustand)

* Не мутировать state напрямую
* Использовать set((state) => ...)
* Разделять бизнес-логику и UI
* Использовать селекторы при необходимости

---

## Components

Перед созданием компонента агент должен определить:

* Это UI компонент → `components/ui`
* Это feature компонент → `features/...`
* Это layout → `components/layout`

Требования:

* Компоненты должны быть переиспользуемыми
* Минимизировать логику внутри UI-компонентов
* Использовать props с типами

---

## Styling

* Использовать только CSS Modules / SCSS
* Не использовать inline-стили без необходимости
* Следовать существующим helper-функциям и mixin'ам

---

## File Structure Awareness

Агент должен:

* Учитывать текущую структуру проекта (`app`, `components`, `features`, `store`, `lib`)
* Не создавать файлы в случайных местах
* Предлагать корректное расположение новых файлов

---

## AI Behavior

Агент должен:

* Сначала думать, потом писать код
* Если задача неоднозначная — предложить варианты
* Не делать предположений о данных без уточнения
* Стараться давать решения, которые можно масштабировать

---

## Forbidden Actions

Агенту запрещено:

* Ломать существующий функционал
* Переписывать большие части кода без запроса
* Игнорировать RULES.MD
* Добавлять ненужные библиотеки
* Использовать устаревшие подходы

---

## Output Format

* Сначала краткое объяснение (если нужно)
* Затем код
* Без лишнего текста
* Код должен быть готов к вставке в проект

---

## Goal

Главная цель агента — помогать разработчику:

* быстрее писать код
* сохранять архитектуру проекта
* избегать ошибок
* развивать проект системно
